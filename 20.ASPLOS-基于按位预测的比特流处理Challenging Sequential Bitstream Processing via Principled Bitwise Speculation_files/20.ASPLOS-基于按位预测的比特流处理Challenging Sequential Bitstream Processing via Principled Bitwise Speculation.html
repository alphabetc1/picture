20.ASPLOS-Challenging Sequential Bitstream Processing via Principled Bitwise Speculation

摘要

现今许多性能关键型应用以位/块为单位并行得处理比特流，以获得更好的性能或更高的空间效率，例如多媒体处理和位图索引。 但是，当并行处理的各个位/块间携带依赖性(dependences)时，整个位流处理将变为串行，这从根本上限制了性能。
在这篇文章里，作者将比特流处理与时序电路进行类比(两者的功能都是进行比特流转换)，通过有限状态机(Finite State Machine，时序电路的基本模型)来建模比特流序列中的依赖关系，最后通过基于lookback(一种FSM并行技术)的按位预测(PBS, Principled Bitwise Speculation)消除比特流处理中的相关性，达到并行处理的效果。

背景

问题定义

比特流处理(bitstream processing)，顾名思义，就是用位运算处理二进制序列。具体定义如下图所示，需要注意的是，输入与输出都可以由多条比特流组成。

比特流处理在现实中有着丰富的应用场景，例如位图索引、模式匹配、图像压缩以及语音解码等。此类应用一般数据量庞大，归功于按位运算的高效性和并行性，理论上我们仍可以收获不错的执行效率。

然而，一个根本问题是，并行处理的几个块之间一旦具有了某种依赖性(bitstream-carried dependences，当前块的处理取决于其他块的结果)，比如下面的加法示例中低位对高位的进位，整个比特流处理就变成了串行，从而限制了整体程序的性能。通常比特流处理的主要功能由代码中的循环体实现(所谓并行也正是实现了循环体内部各轮数之间的并行)，因此这种依赖又被称作循环间依赖。

现状

现有的针对比特流处理的优化主要集中在使用SIMD intrinsics函数[1]进行细粒度并行。尽管性能有所提高，仍有一些问题阻碍了基于SIMD的优化的生产率和效率。

  • 首先，使用SIMD intrinsics函数编程异常困难。
    以下面的LBSAdd(long bitstream add)为例，它实现了两个任意长度比特流的以字节为单位(而不是传统的以位为单位)的加法，其复杂程度可见一斑。
  • 其次，SIMD intrinsics函数只是提供一种并行化处理的方法，在这里数据间的依赖性只是被修改而绝不会被消除，比如在下图中，随着加法的基本单位由位变为字节，依赖的基本单位也随之改变。
  • 最后，有些比特流一次需要处理的位数过于庞大，比如现有的长比特流加法(long bitstream addition)最多可以累加到4096位，均分到每个核后显然会超过SIMD数据宽度[2]，也就难以实现并行处理。

方法

作者并没有从正面解决这个问题，而是巧妙得将这个问题转化为某个已解决的问题，通过旧问题解决新问题。
下图是实现比特流处理的代码中循环体和时序电路的对比图，可以看出，两者的功能同样是转换比特流序列，同时时序电路中的存储器类似于比特流程序中的循环携带依赖关系，这些依赖关系由程序结构隐式决定。假如我们可以构建出能表达数据间依赖关系的有限状态机(FSM)，就能通过一种并行的FSM计算方法消除数据间的依赖性。

接下来需要思考的，就是如何把比特流处理程序转换为FSM模型。

那么如何将比特流程序转换为FSM模型呢，其实说来也简单，只需要三步：
　1. 打开冰箱门；

  1. 找出所有的依赖位(Dependent Bits，造成数据间依赖性的位)；
  2. 生成真值表；
  3. 将真值表转换为FSM。


在下面的实现环节，我们将介绍每一步的步骤及其相应算法。

实现

step1：找出所有的依赖位

首先我们先观察一下这些位具有哪些特点

  • 特点一：读后写，或者说先使用后修改
    参考下面右图中的变量a，当前轮a的取值取决于上一轮，这也就造成了数据依赖。

    算法：
    首先维护一个集合S，集合内部数据以(变量：程序段)标注，接着从后往前扫描整个循环体，每当读一个变量，将这个变量及其程序段写入集合，而每当写一个变量，就从集合中将其删去。

  • 特点二：有些数的某些位从来不会被使用，这些位就应该被排除在依赖位之外
    下图中的变量b与7(00000111)相与，那么b的前五位就不会对运算结果有任何影响，也就不会造成数据依赖。

    要做到这一步，系统还需要始终记录常量位，比如下面变量a的第3-7位已知为0且不会改变，那么相应的，d的第3-7位也不会造成数据依赖。

    算法：
    用四种符号标注所有位：unknown(未知)、0(常量0)、1(常量1)、可排除(非依赖位)
    首先将所有位的符号标注为unknown，然后进行第一轮扫描(常量位检查，方向是从前到后)，比如扫描到L3：bu = c & 1，那么bu的前7位就都标注为0，依此类推。接着进行第二轮扫描(可排除位检查，方向是从后到前)，一旦扫描到某个数与常量或者常数位操作，进行相应的标注。

  • 特点三：不应该是只读的
    如果一个变量在循环体外已经声明，而在整个循环体内部其值没有改变，则一定不会产生数据依赖，这样的数也可以排除在依赖位之外。

    算法：
    只读检查的算法比较简单粗暴，先是将所有位都标注为只读，再扫描一遍程序，每当写一个变量就将该变量的所有位设置为非只读。

通过整合以上几点(读后写、排除无效位、非只读)，我们只要将存在读后写的位和未知位取个交集，再从中减去所有只读位，就能找到所有的依赖位。

step2：生成真值表

至此我们已经得到了比特流处理中的各个依赖位，当前轮循环只要得到这几个位的值就可以继续处理，换句话说，如果我们能以某种方式“猜”出这几个位的值，数据间的依赖性随即迎刃而解，我们的程序也就能够并行处理了。有限状态机的状态正是由这些依赖位的不同取值构成的，n个位可以构成个状态。

下面是真值表的构造方法
我们将所有依赖位(当前例子里是C[0:3])和输入排列组合，并产生相应的下一阶段状态位及输出。

  • 下一阶段状态位C'[0:3] ：将C与输入进行位运算后取C的位数；
  • 输出：将C与输入进行位运算后取输入的位数；

step3： 将真值表转换为FSM

构建完真值表后，我们以依赖位的各种取值为状态，以输入为条件构建FSM。
注：CS(当前状态)、I(输入)、NS(下一阶段状态)、O(输出)

到这里我们终于将比特流处理的代码转化成了FSM，然而此时的FSM仍旧是个串行执行的时序电路，要想并行处理，就必须知道任意时刻所对应的FSM状态，那么如何得到某一时刻的FSM状态呢？
答案其实非常简单：猜。
当然，要猜也得是有原则有组织有计划地猜，否则这篇文章怎么能发ASPLOS呢？
下面介绍这篇文章的核心科技：基于lookback的按位预测。

基于lookback的按位预测(principled bitwise speculation)

刚才也说了，FSM的状态是由依赖位的不同取值构成的，假如依赖位的个数为n，那么任一时刻正确的状态一定在这个状态中(傻了吧，就是这么简单粗暴)。
虽然我们不知道任何时刻的FSM状态(除了初始时刻)，但是我们知道所有时刻的输入(就是被处理的比特流序列)，换句话说，只要知道了一个时刻的状态，其后的任一时刻状态都可以算出来。所谓的并行处理，就是根据可用的CPU内核数均匀划分输入序列，然后将每个输入分区分配给一个线程进行处理。 除第一个线程外，所有线程都是推测式运行的。为了猜出开始状态，一个推测式线程会代入各个状态并算出它们的结束状态，再选次数最多的作为预测结果。最后将当前轮初始状态与上一轮结束状态相比较确定有没有猜对。


下面是一个预测示例，在partition1中，开始所有状态(s1，s2，s3)经过7位比特流输入(0010101)后都变为s1，可见选择任一状态作为开始其效果都是等同的。

这种推测式执行的一个弊端便是，假如某一片段的开始状态猜错了，当前段就必须以正确开始状态重新计算得到结束状态，以验证下一片段的开始状态是否正确，这无疑大大增加了时间损耗。在下面的优化环节我们将介绍一种免于重新计算的快速修正技术。

Reference

[1]SIMD intrinsics:
采用 SSE 进行编程的方式有两种：一种是直接编写汇编指令代码, 另一种是使用 intrinsics 函数编程(外表像是普通函数，但实际上会被 compiler 直接编译成组合语言)。使用 intrinsics 而非汇编指令编程是一种更加直接的方法，与标准 C/C++ 编程类似。此外，使用 intrinsics 编程还有助于编译器更好地优化代码。

[2]SIMD数据宽度/SIMD lane:
SIMD就用一条语句并行处理多个数据。被并行处理的每个数据就是一个lane，数据的个数即为lane的个数，数据的总长度就是SIMD数据宽度。打个比方来说，一条高速公路上的车道就是lane，车道数就是lane的个数，有多少lane就表示有多少车可以并排同时开。SSE的128bit或者AVX的512bit是一条命令可以处理的数据总宽度，也就相当于高速公路的宽度。高速公路上有几车道，除了与高速公路的宽度有关，还与车道的宽度有关。将高速公路的宽度除以车道的宽度，就可以计算出车道数。类似的，将SIMD命令的数据宽度（如128bit，512bit）除以该命令处理的单个数据的宽度（如32bit、或者64bit），就是lane的个数。比如当我们的SIMD命令执行在32bit模式下的时候，那么128bit的SSE命令就是4 lane；而AVX512就是16 lane。也就是一条SSE命令就能完成一个4维向量的四则运算而一条AVX512命令就能完成一个4x4矩阵的某些运算。
